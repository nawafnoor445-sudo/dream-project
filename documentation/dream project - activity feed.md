Dream Project — Activity Feed (V1)
الإصدار: V1.0 (مبدئي)
التاريخ: 2026-02-08

1) الهدف
سجل أحداث واضح يعرض ما يحدث داخل المشروع أو على مستوى المنصة بهدف الشفافية
والمتابعة دون تفاصيل تقنية حساسة.

2) النطاق (Scope)
- Project Activity: يظهر لأعضاء المشروع وصاحبه.
- Platform Activity: يظهر في الصفحة العامة/Explore أو للمدير حسب النوع.

3) أنواع الأحداث (Project Activity)
- إنشاء مشروع
- تغيير حالة المشروع
- إرسال طلب انضمام
- قبول/رفض طلب انضمام
- انسحاب عضو
- إخراج عضو
- إنشاء مهمة
- تغيير حالة مهمة (In Progress / In Review / Completed)
- تعيين مهمة لمطور
- تمديد وقت مهمة
- إضافة قسم مخصص
- إضافة رابط لقسم

4) أنواع الأحداث (Platform Activity)
- أخبار المنصة الجديدة (Admin)
- إجراءات إدارية كبيرة (حذف مشروع، حظر)

5) الحقول الأساسية لكل حدث
- scope: project | platform
- project_id (nullable عند platform)
- actor_user_id (nullable لبعض أحداث الإدارة)
- event_type (string ثابت)
- summary (نص مختصر قابل للعرض)
- created_at

6) قواعد التسجيل
- كل تغيير حالة مشروع أو مهمة يولد حدثًا تلقائيًا.
- أي عملية عضوية (طلب/قبول/رفض/سحب/إخراج) تولد حدثًا.
- إضافة روابط أو أقسام تولد حدثًا.
- أحداث الإدارة تظهر فقط كـ Platform Activity.

7) الخصوصية
- أحداث المشروع لا تظهر لغير أعضاء المشروع.
- لا يتم عرض بيانات حساسة (بريد، أسباب داخلية تفصيلية).
