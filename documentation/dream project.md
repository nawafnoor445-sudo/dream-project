Dream Project — الوثيقة الأساسية
الإصدار: V1.1 (مبدئي)
التاريخ: 2026-02-08

1) نظرة عامة
Dream Project منصة اجتماعية احترافية لعرض المشاريع التقنية أثناء بنائها وربط
أصحاب المشاريع بالمطورين. المنصة ليست بيئة تطوير ولا تستضيف الكود أو الملفات؛
هي لوحة تنظيم، متابعة، وروابط خارجية للمصادر. الهدف هو جعل المشاريع قابلة
للاكتشاف والتقييم أثناء التطوير وليس بعد الانتهاء.

2) الأهداف
- تمكين أصحاب المشاريع من عرض أفكارهم وبناء فرق.
- تمكين المطورين من الانضمام لمشاريع حقيقية لتطوير المهارات أو العمل خارج المنصة.
- تقليل المشاريع الوهمية وعدم الالتزام عبر نظام ثقة داخلي.
- جعل المشاريع قابلة للاكتشاف والتجربة عبر روابط خارجية.

3) أدوار المستخدمين
- Guest: عرض المشاريع والأيقونات والوصف فقط.
- Developer: ملف شخصي، مهارات، مستوى ثقة، يمكنه طلب الانضمام.
- Project Owner: إنشاء وإدارة المشاريع والأقسام والمهام والعضوية.
- Admin: إدارة المخالفات، التحذيرات، حذف المشاريع، الحظر، أخبار المنصة.

4) الكيانات الأساسية
- User: الاسم، البريد، الصورة، المهارات، مستوى الثقة، الدور، تاريخ التسجيل.
- Project: المالك، الاسم، الوصف، عدد الفريق المطلوب والحالي، الحالة، التواريخ،
  الإصدار الحالي للفكرة.
- Section: اسم القسم، نوع الصلاحية الافتراضية، مرتبط بالمشروع.
- Task: العنوان، الوصف، الحالة، المطور المعين، تاريخ الاستحقاق.
- MembershipRequest: المستخدم، المشروع، الحالة، تاريخ الطلب.
- SectionPermission: المستخدم، القسم، صلاحيات عرض أو عمل.
- TrustEvents: سجل داخلي لزيادات وخصومات الثقة.
- Notification: المستخدم، النوع، الرسالة، حالة القراءة، التاريخ.
- DisputeLog: المشروع، المستخدم، نوع المخالفة، الوصف، حالة الحل، التاريخ.
- ActivityFeed: سجل الأحداث العامة داخل المشروع والمنصة.

5) حالات المشروع وقواعد الانتقال
الحالات: Planned → Team Building → In Progress → Ready for Testing → Completed
- الانتقال يتم فقط من صاحب المشروع.
- In Progress بعد اكتمال الفريق أو قبول عدد كاف من المطورين.
- Ready for Testing بعد اكتمال جميع المهام الأساسية.
- Completed بعد انتهاء مرحلة الاختبار.

6) الأقسام داخل المشروع
أقسام افتراضية في V1: Backend, Frontend, Design, Architecture
- يمكن لصاحب المشروع إضافة أقسام مخصصة باسم فقط بدون منطق إضافي.
- لكل قسم قائمة روابط خاصة به (GitHub, Figma, Discord وغيرها).

7) المهام
حالات المهام: Not Started, In Progress, In Review, Completed
- كل مهمة مرتبطة بمطور واحد فقط في V1.
- المطور ينقل المهمة إلى In Review.
- صاحب المشروع فقط ينقل المهمة إلى Completed.
- الإنجاز يُحسب للتقييم فقط عند Completed.
- يمكن لصاحب المشروع إضافة وقت إضافي للمهمة.

8) الصلاحيات
- الصلاحيات على مستوى القسم فقط.
- افتراضيًا: عرض + عمل.
- قسم Architecture افتراضيًا عرض فقط ويمكن تغييره.
- الصلاحيات تحدد عند الانضمام ويمكن تعديلها لاحقًا من صاحب المشروع.

9) العضوية وطلبات الانضمام
- الموافقة من صاحب المشروع فقط.
- المالك لا يُحسب ضمن عدد أعضاء الفريق.
- حد أقصى 10 طلبات انضمام نشطة لكل مطور.
- يمكن للمطور سحب طلبه.
- إخراج عضو يتطلب سبب واضح ويؤثر على تقييم صاحب المشروع إذا لم يوجد سبب مبرر.

10) نظام الثقة والتقييم
- نظام الثقة داخلي ويعرض للمستخدم كمستوى (Bronze, Silver, Gold).
- لكل مهمة مكتملة: زيادة تلقائية تحددها المنصة لاحقًا.
- إنجاز المهمة في الوقت المحدد يسمح لصاحب المشروع بإضافة تقييم إيجابي.
- التأخير يؤدي لخصم تلقائي من تقييم المطور.
- الانسحاب يؤدي لخصم دائم من تقييم المطور.
- إخراج عضو بدون سبب واضح يؤدي لخصم من تقييم صاحب المشروع.
- التقييم النهائي للمطور = حساب المنصة + تعديل صاحب المشروع.
- تقييم المشروع يُحسب تلقائيًا من نسبة اكتمال المهام واستقرار الفريق.

11) المخالفات والعقوبات
- المراجعة يدوية من Admin فقط في V1.
- تحذير أول، تحذير ثانٍ، ثم إجراءات إدارية (حذف مشروع أو حظر).
- لا يوجد حظر تلقائي بدون مراجعة بشرية.
- سجل المخالفات مرئي للمنصة وصاحب الحساب فقط.

12) الإشعارات (V1)
- داخل التطبيق فقط في الإصدار الأول.
- أحداث: طلب انضمام، قبول أو رفض، انسحاب عضو، تغيير حالة المشروع، اكتمال
  مهمة، مخالفات أو تحذيرات.

13) الاكتشاف (Discovery)
- فلاتر: حالة المشروع، الأقسام المطلوبة، عدد الفريق، الأحدث.
- الأحدث حسب تاريخ الإنشاء.
- قسم Trending Projects يعتمد على عدد الطلبات أو الزيارات.

14) النطاق (V1 Scope)
ما يوجد: عرض المشاريع، العضويات، الأقسام والمهام، روابط خارجية، نظام ثقة،
Activity Feed.
ما لا يوجد: استضافة كود، تشغيل التطبيقات داخل المنصة، بيع أو مدفوعات، دردشة
داخلية.

15) اتجاهات مستقبلية (V2+)
- إشعارات مخصصة حسب التفضيلات.
- نظام تقييم تفصيلي بالأرقام داخلي فقط.
- Marketplace داخلي.
- دردشة مرتبطة بالمشاريع والمهام.
- مشاريع خاصة (Private Projects).

16) ترتيب التنفيذ
1) Users + Auth + Profile + Skills + TrustLevel
2) Projects + Sections الافتراضية + Custom Sections
3) Join Requests + Membership + Section Permissions
4) Tasks
5) Trust Events
6) Notifications
7) Activity Feed
